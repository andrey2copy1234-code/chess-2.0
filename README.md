# chess-2.0
проект шахмат. с ботом и многим другим - 2 версия. имеет сохранения и полностью совместим с chess 1.0 и багов мало. есть возможность играть по сети и локально и глобально
## Запуск
Скачайте с releases .zip файл и запустите файл chess.exe
## Управление
на кнопку b запускается бот

на кнопку s включается/выключается переворот экрана

на кнопку a запускается анализ ввиде графиков где показывается как вы сыграли партию

на кнопку d изменить глубину просчёта бота в полуходах (один полуход примерно +200 эло)

на горячую клавишу m выбор режима

на Control-z отменить ход (не работает в игре по сети)

на Control-Shift-z отменить отменение хода (не работает в игре по сети)
## Как играть по сети?
Если вы играете в локальной сети:
1. Выберете играть "игрок против игрока онлайн"
2. тот кто играет за чёрных выбирает "Запустить сервер" а тот кто за чёрных "Подключится к серверу"
3. тот кто выбрал подключится к серверу вводит local address который написало у того кто нажал "Запустить сервер"
4. Играйте!
5. Вы можете открыть сохранение и оно откроется у обоих (тоже самое с restart)

Если вы играете не в локальной сети:
1. Выберете играть "игрок против игрока онлайн"
2. тот кто играет за чёрных выбирает "Запустить сервер" а тот кто за чёрных "Подключится к серверу"
3. тот кто выбрал подключится к серверу вводит global address который написало у того кто нажал "Запустить сервер"
4. Играйте!
5. Вы можете открыть сохранение и оно откроется у обоих (тоже самое с restart)

Если у вас не получилось то проверьте консоль:
* если там написано где-то WinAPI UPnP и второй раз UPnP то это проблемы с роутером. иногда он просто не поддерживает а иногда это вызвано закрытием игры по сети диспечером задач. в таком случае необходимо перезагрузить роутер.
* если WinAPI UPnP и HRESULT то гуглите в интернете код который написан (он не относится к самим шахматам) и решайте проблему как сказано в интернете.
## Бот
Работает на minimax с альфа-бета отсечением и оценочной функции ввиде цена всех фигур моих-цена всех фигур врага. оценка происходит прямо во время расчёта (на всех глубинах) а не когда уже depth=1.
## Шахматный движок
Использует int8_t[8][8] для хранения фигур где +фигуры это белые -фигуры это чёрные а то насколько +n фигура или -n фигура это её цена (не повторяется). вот таблица цен:
 Имя фигуры | abs(f) | Индификатор в файле .cmap | Именование в коде |
| :--- | :---: | :---: | ---: |
| Пустота | 0 | - | `figureType::Empty` |
| Пешка | 1 | `пешка` | `figureType::Pawn` |
| Конь | 3 | `конь` | `figureType::Knight` |
| Слон | 4 | `слон` | `figureType::Bishop` |
| Ладья | 4 | `лодья` | `figureType::Rook` |
| Ферзь | 9 | `ферзь` | `figureType::Queen` |
| Король | 8 | `король` | `figureType::King` |
Как видно король тут 40 но фактически он не используется в оценке в случае взятия короля считается счёт по формуле 900000+100*depth+score где depth это от максимальной глубины просчёта бота (сейчас 6) до 1 где 1 это конец.
## Сохранения
### .txt или .cmap
Это старые сохранения. сохраняют только доску.

Этот движок может использовать сохранения формата chess 1.0. вот как обозначается в нём одна фигура: `name posx posy steps color ` там где posx и posy указаны от 1.0 это левый верхний угол и всегда с .0 
также с steps а color указывается как True (белый) или False (чёрный). все фигуры пишутся без пробелов (ну точнее через пробел но я указал туда доп пробел раньше). в самом конце файла дописывается `ccolor ` где ccolor это кто сейчас ходит где есть True (белый) и False (чёрный). 
### .ctmap
Новое сохранение. сохраняет историю ходов. полностью не совместим с .txt/.cmap.

Данное сохранение использует бинарный формат данных где первые 4 байта хранят 4 байтный unsigned int формата Little-endian. дальше идёт какой ход совершила фигура. структура хода соответствует этой структуре. выравнивание по 1 байту:
```cpp
struct step {
 int8_t f;  // фигура которая совершила ход
 int8_t af; // фигура которую съели при ходе
 int8_t rf; // фигура на которую заменяется при ходе старая фигура
 uint8_t stx; // из какой клетки начался ход (0 это правая клетка от лица белых)
 uint8_t sty; // из какой клетки начался ход (0 это клетка на стороне белых)
 uint8_t ex;  // в какую клетку ход
 uint8_t ey;  // в какую клетку ход
};
```
все int8_t заполняются числами  где + это белые а - это чёрные. сами id ищите выше в пункте о шахматном движке.
